## 实施阶段规划

### 阶段 A — 布局还原 & 视觉拉齐（纯前端，不依赖后端）

**预计工期**：1~2 周
**目标**：页面"看起来"达到设计稿 70~80% 的视觉效果

#### 输入页
- [ ] 风景 hero 背景图（静态图 + 渐变遮罩）
- [ ] 热门目的地快选 chips
- [ ] DatePicker 日期选择器组件（前端先做，可传也可不传后端）
- [ ] BudgetSlider 预算范围滑块（同上）
- [ ] 节奏改为连续滑块样式
- [ ] 手写体装饰文字

#### 结果页
- [ ] 三栏布局改造（左侧栏 + 主内容区）
- [ ] 卡片加城市占位封面图（前端城市 → 图片映射）
- [ ] "调整偏好" 按钮
- [ ] 底部 AI 生成声明
- [ ] 收藏按钮（UI 占位，功能待 C 阶段）

#### 详情页
- [ ] 三栏布局（左侧栏 + 中间时间轴 + 右侧地图）
- [ ] 左侧栏占位区域（预算/天气区域先用 mock 数据或 skeleton）
- [ ] 今日行程概览统计（前端从已有数据汇总）
- [ ] 地图/卫星切换按钮
- [ ] Day 标签增加日期显示

#### Header
- [ ] 完整导航栏（链接暂不跳转或跳 404）

#### 通用
- [ ] 卡片封面图组件（带 fallback）
- [ ] 侧边栏布局组件

---

### 阶段 B — 数据填充（需后端 API 配合）

**预计工期**：2~3 周
**目标**：数据从后端真实填充，达到设计稿 90%+ 还原

#### 后端需返回的新数据

| 数据 | 对应后端改动 | 影响页面 |
|------|------------|---------|
| `budget` 分类预算（交通/住宿/餐饮/门票/其他） | Result API 增加 budget 字段 | 结果页、详情页 |
| `cover_image_url` 城市/方案封面图 | POI 图片或城市图库 | 结果页 |
| `place.image_url` 景点照片 | POI 详情增加图片 | 详情页 |
| `place.start_time` / `place.end_time` 时间段 | 行程规划输出时间 | 详情页 |
| `place.tags` 景点标签 | POI 详情增加标签 | 详情页 |
| `place.rating` 景点评级（5A/4A） | POI 详情增加评级 | 详情页 |
| `commute_leg.encoded_polyline` 真实路线 | 高德路线 API 缓存 | 详情页地图 |
| `start_date` / `end_date` 行程日期 | 接收并返回起止日期 | 输入页、详情页 |
| 天气数据 | 天气 API 集成（和风天气/OpenWeather） | 结果页、详情页 |

#### 前端任务
- [ ] 多城市选择组件（chips + 搜索）
- [ ] 预算明细侧栏组件（进度条 + 分类）
- [ ] 天气预报组件（当前 + 多日预报）
- [ ] 景点卡片升级（图片 + 时间 + 标签 + 评级）
- [ ] 真实路线 polyline 渲染（替换虚线）
- [ ] 导出 PDF/Excel 功能
- [ ] 封面图从后端 URL 渲染

---
